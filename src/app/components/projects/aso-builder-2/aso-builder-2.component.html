ASOBuilder 2.0

ASOBuilder 2.0 es la suguinte version del projecto ASOBuilder, que pretende agilizar el desarrollo de APIs; generando código Java, el cual cumpla con las normativas y estándares requeridos por el cliente, liberando al desarrollador de trabajo monótono y repetitivo, permitiendo que este se enfoque solo en aquellos apartados que requieran supervisión humana.

Tecnologías:

Java 100%
Spring Boot 100%
Thymeleaf 100%

Propuesta y desarrollo: Miguel Villa

Año: 2021

Cliente: cliente no disponible

Problemática

Las APIs requeridas por el cliente se describen tanto en RAML como en archivos de Excel (insumos), en los cuales se indica tanto la presentación, datos de entrada y salida, así como el servicio el cual deben consumir (backend). Es a partir de estos insumos que el desarrollador deberá programar el API, sin embargo este proceso suele ser tardado y complejo, por una parte debido a la magnitud de entradas y salidas, y por otra parte debido a errores comunes que surgen durante el desarrollo, (errores de tipado, malas prácticas de programación).

La descripción completa del proyecto se encuentra en ASOBuilder, esta versión se considera uno de los trabajos a futuros que se plantean durante el análisis del proyecto previo ASOBuilder.

En esta versión se realizan ajustes mínimos en cuanto al código generado ya que desde la primera versión desarrollada en 2019 los estándares implementados en el código han cambiado. Asimismo se realizó un cambio en la interfaz principal la cual ahora hace uso de un editor de código que permite capturar la descripción del proyecto en formato YAML y previsualizar el código generado, a diferencia del proyecto inicial que capturaba la información en un formulario web y que solo permitía ver el código hasta que se descargaba el archivo ZIP.
